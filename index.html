The pointer CSS media feature tests whether the user has a pointing device (such as a mouse), and if so, how accurate the primary pointing device is.

The pointer media query has three values:

  1. none - device has no pointer
  2. coarse - pointer has limited accuracy
  3. fine - pointer has high level of accuracy

Example

This example creates a small checkbox and radio button for users with fine primary pointers and a large checkbox for users with coarse primary pointers.

The form controls below will be enlarged for devices with a coarse pointer

      
        
        
      
    
      
        @media (pointer:coarse) {
          input[type="checkbox"], input[type="radio"] {
            min-width:30px;
            min-height:40px;
            background:transparent;
          }
        }

The any-pointer CSS media feature tests whether the user has any pointing device (such as a mouse), and if so, how accurate it is.

The any-pointer media query has three values:

  1. none - No pointing device is available.
  2. coarse - At least one input mechanism includes a pointing device of limited accuracy.
  3. fine - At least one input mechanism includes an accurate pointing device.

Example

This example creates a small checkbox and radio button for users with fine primary pointers and a large checkbox for users with coarse primary pointers.

The form controls below will be enlarged for devices with a coarse pointer

      
        
        
      
    
      
        @media (any-pointer:coarse) {
          input[type="checkbox"], input[type="radio"] {
            min-width:30px;
            min-height:40px;
            background:transparent;
          }
        }

The hover CSS media feature can be used to test whether the user's primary input mechanism can hover over elements.

The hover media query has two values:

  1. none - The primary input mechanism cannot hover at all or cannot conveniently hover (e.g., many mobile devices emulate hovering when the user performs an inconvenient long tap), or there is no primary pointing input mechanism.
  2. hover - The primary input mechanism can conveniently hover over elements.

Example

This example creates a small checkbox and radio button for users with fine primary pointers and a large checkbox for users with coarse, primary pointers.

The form controls below will be enlarged for devices with a coarse pointer

      
         Look at me!
      
    
      
        @media (hover: hover) {
          a:hover {
            background: yellow;
          }
        }

The any-hover CSS media feature can be used to test whether any available input mechanism can hover over elements.

The hover media query has two values:

  1. none - None of the available input mechanism(s) can hover conveniently, or there is no pointing input mechanism.
  2. hover - One or more available input mechanisms can conveniently hover over elements.

Example

      
         Look at me!
      
    
      
        @media (any-hover: hover) {
          a:hover {
            background: yellow;
          }
        }

The color-gamut CSS media feature can be used to test the approximate range of colors that are supported by the user agent and the output device.

The hover media query has three values:

  1. srgb - The output device can support approximately the sRGB gamut or more. This includes the vast majority of color displays.
  2. p3 - The output device can support approximately the gamut specified by the DCI P3 Color Space or more. The p3 gamut is larger than and includes the srgb gamut.
  3. rec2020 - The output device can support approximately the gamut specified by the ITU-R Recommendation BT.2020 Color Space or more. The rec2020 gamut is larger than and includes the p3 gamut.

Example

      
        
This is a test.

      
    
            
              @media (color-gamut: srgb) {
                p {
                  background: #f4ae8a;
                }
              }

The overflow-inline CSS media feature can be used to test how the output device handles content that overflows the initial containing block along the inline axis.

The overflow-inline media query has three values:

  1. none - Content that overflows the inline axis is not displayed.
  2. scroll - Content that overflows the inline axis can be seen by scrolling to it.

Example

      
        
Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nullam ac turpis eleifend, fringilla velit ac, aliquam tellus. Sed nec augue congue eros accumsan tincidunt sed eget ex.

      
    
            
              @media (overflow-inline: scroll) {
                p {

                }
              }

The overflow-block CSS media feature can be used to test how the output device handles content that overflows the initial containing block along the block axis.

The hover media query has four values:

  1. none - Content that overflows the block axis is not displayed.
  2. scroll - Content that overflows the block axis can be seen by scrolling to it.
  3. optional-paged - Content that overflows the block axis can be seen by scrolling to it, but page breaks can be manually triggered (such as via break-inside, etc.) to cause the following content to display on the following page.
  4. paged - Content is broken up into discrete pages; content that overflows one page in the block axis is displayed on the following page.

Example

      
        
This is a test.

      
    
            
              @media (overflow-block: scroll) {
                p {
                  color: red;
                }
              }

The update CSS media feature can be used to test how frequently (if at all) the output device is able to modify the appearance of content.

The hover media query has three values:

  1. none - Once it has been rendered, the layout can no longer be updated. Example: documents printed on paper.
  2. slow - The layout may change dynamically according to the usual rules of CSS, but the output device is not able to render or display changes quickly enough for them to be perceived as a smooth animation. Examples: e-book readers or severely underpowered devices.
  3. fast - The layout may change dynamically according to the usual rules of CSS, and the output device is not unusually constrained in speed, so regularly-updating things like CSS Animations can be used. Example: computer screens.

Example

      
        
If this text animates for you, you are using a fast-updating device.

      
    
            
              @keyframes jiggle {
                from {
                  transform: translateY(0);
                }

                to {
                  transform: translateY(25px);
                }
              }

              @media (update: fast) {
                p {
                  animation: 1s jiggle linear alternate infinite;
                }
              }
